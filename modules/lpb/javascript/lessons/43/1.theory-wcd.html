1. Introduction

Les "browser events" (événements du navigateur) sont des actions ou des interactions qui se produisent dans une page web, comme un clic de souris, une pression sur une touche du clavier, ou le chargement d'une page. JavaScript permet de détecter ces événements et d'exécuter du code en réponse à ceux-ci. C'est une partie essentielle pour rendre les pages web interactives.

2. Qu'est-ce qu'un événement ?

Un événement est un signal envoyé par le navigateur lorsqu'une action spécifique se produit.
Par exemple :

  • L'utilisateur clique sur un bouton (click).
  • Une page finit de se charger (load).
  • L'utilisateur tape sur une touche (keydown).

3. Comment écouter un événement ?

Pour réagir à un événement, on utilise la méthode addEventListener sur un élément HTML.
Voici la syntaxe de base :

  • element : L'élément HTML sur lequel on veut écouter l'événement (ex. un bouton, un paragraphe).
  • 'nomDeLEvénement' : Le type d'événement (ex. 'click', 'mouseover', 'input').
  • fonctionÀExécuter : La fonction qui sera appelée quand l'événement se produit.

6. Quelques événements courants

  • click : Clic sur un élément.
  • mouseover : La souris passe au-dessus d’un élément.
  • mouseout : La souris quitte un élément.
  • keydown : Une touche est enfoncée.
  • submit : Un formulaire est soumis.
  • load : La page ou une ressource a fini de charger.

9. Bonnes pratiques

  • Évitez d’écrire trop de logique dans l’écouteur, préférez appeler une fonction séparée.
  • Testez vos événements dans la console pour vérifier qu’ils fonctionnent comme prévu.
  • Pensez à la compatibilité entre navigateurs pour des événements plus rares.
