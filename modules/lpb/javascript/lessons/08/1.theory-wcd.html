En Javascript, les types de valeurs null et undefined sont deux valeurs primitives. Elles sont un peu particuliers car ils ne contiennent qu’une valeur chacun : les valeurs null et undefined.

La valeur null

La valeur null correspond à l’absence de valeur ou du moins à l’absence de valeur connue. Pour qu’une variable contienne null, il va falloir stocker cette valeur qui représente donc l’absence de valeur de manière explicite.
La valeur null va être utile dans certains cas où on souhaite explicitement indiquer une absence de valeur connue.
Contrairement à undefined, null est intentionnel et signifie explicitement "aucune valeur".

undefined

Une variable est considérée comme undefined lorsqu'elle est déclarée mais n’a pas encore reçu de valeur.

Caractéristiques :
  • C'est le type par défaut d'une variable non initialisée.
  • Si une fonction n’a pas de retour explicite, elle retourne undefined.
  • Une propriété d’un objet qui n'existe pas ou une valeur d'un tableau hors limites retourne également undefined.

Comparaisons importantes

1. undefined vs null (égalité lâche) :
Avec ==, Javascript considère que les deux valeurs sont équivalentes.

2. undefined vs null (égalité stricte) :
Avec ===, les types étant différents (undefined et object), ils ne sont pas strictement égaux.

Quand utiliser quoi ?

  • undefined : Laissez Javascript l'utiliser par défaut pour signaler une absence involontaire.
  • null : Utilisez-le intentionnellement pour indiquer qu’une variable est vide ou sans valeur.

En résumé, undefined est implicite et lié au comportement par défaut de Javascript, tandis que null est explicite et signifie l'absence voulue d'une valeur.
