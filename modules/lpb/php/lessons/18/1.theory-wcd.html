I. Présentation de l'instruction return

Dans cette leçon, nous allons apprendre la fonctionnalité de la structure de contrôle return et apprendre à l’utiliser pour retourner le résultat d’une fonction.

Jusqu’à présent, le seul moyen que nous avions de faire remonter le résultat de nos fonctions était d’afficher ce résultat en utilisant un echo dans le code de la fonction.
Mais, si en effet il y a des fonctions qui n’ont pas besoin de retourner de résultat, elles exécutent du code qui effectue un traitement, il est souvent nécessaire de récupérer le résultat d’une fonction pour l’utiliser dans le reste du code.

C'est la structure de contrôle nommée return qui va nous permettre, dans le code d'une fonction, de retourner un résultat qui va ensuite être stocké dans une variable ou autre pour être manipulé. Comme vous devez en avoir l'intuition maintenant, cela va nous être très utile pour la suite de notre apprentissage et surtout en tant que développeur dans l'élaboration de nos algorithmes.

IMPORTANT : l’instruction return va terminer l’exécution d’une fonction, ce qui signifie que l'on placera généralement cette instruction à la fin d'une fonction puisque le code qui suit une instruction return dans une fonction ne sera jamais interprèté.

Exemple : 

      
        function addition($a, $b) {
            $result = $a + $b;
            return $result;
            echo "Je ne serai jamais affiché";
        }

Dans cet exemple, la fonction addition va additionner les deux paramètres $a et $b et retourner le résultat. L’instruction echo qui suit l’instruction return ne sera jamais exécutée.

Pour l'instant notre fonction d'addition était déclarée comme suit : 

    function addition($a, $b) {
        $result = $a + $b;
        echo $result;
    }

Mais maintenant, nous allons pouvoir récupérer le résultat de cette fonction pour l'utiliser dans le reste de notre code. avec l'instruction return et écrire notre fonction ainsi : 

    function addition($a, $b) {
        $result = $a + $b;
        return $result;
    }

Mettons cela en pratique dans l'Exo1 du code source.

Comme nous pouvons le constater et comme prévu, l'appel de la fonction add() affiche le résultat de l'addition des deux paramètres passé en argument.
Lors de l'appel de la fonction add2() rien ne s'affiche, car le résultat de l'addition est retourné par la fonction. Par la suite nous pouvons stocker ce résultat dans une variable pour ensuite le manipuler ou l'afficher. Nous autions pu également l'afficher directement en utilisant un echo lors de l'appel de la fonction.

II. La déclaration des types de valeurs de retour

Depuis PHP7, nous pouvons également de façon similaire à la déclaration de type pour les arguments pouvoir déclarer le type des valeurs de retour. Nous allons utiliser les mêmes types que ceux que nous avons abordés dans les leçons précédentes.

NOTE : Le typage strict affecte également les types de retour. Si celui-ci est activé, la valeur retournée doit être du type attendu. Dans le cas contraire, une exception (erreur) sera levée par l'interprèteur PHP.

Exemple : 
    function addition(int $a, int $b) : int {
        $result = $a + $b;
        return $result;
    }

Dans le code ci-dessus notre fonction addition prend en paramètre deux entiers et retourne un entier. Si nous activons le typage strict, nous devons respecter ce typage. A contrario, dans le cas où le typage strict n’est pas activé (qui est le cas par défaut), si les valeurs retournées ne sont pas du type attendu alors elles seront transtypées, ce qui signifie que le PHP va essayer de transformer leur type pour qu’il corresponde au type attendu.
