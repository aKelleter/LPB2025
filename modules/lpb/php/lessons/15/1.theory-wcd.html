Présentation

Dans cette nouvelle leçon, nous allons aborder 4 nouvelles structures de contrôle qui vont se révéler très utiles et pratiques lorsqu’on voudra créer un site : include, require, include_once et require_once.

Ces structures permettent d’inclure un fichier dans un autre. Cela peut être très utile pour réutiliser du code, séparer le code en plusieurs fichiers, etc. Les structures include et require vont nous permettre toutes deux d’inclure des fichiers de code (ou plus exactement le contenu de ces fichiers) à l’intérieur d’autres fichiers de code. La différence entre les deux est que si le fichier inclus n’est pas trouvé, include va simplement afficher un warning et continuer l’exécution du script, alors que require va afficher une erreur fatale et arrêter l’exécution du script.

Exemples d'utilisation

Sur un site web ou une application, il y a de fortes chances que l’en-tête, le menu et le pied de page soient identiques pour toutes les pages.
Plutôt que de réécrire tout le code relatif à ces différentes parties sur chacune de nos pages, pourquoi ne pas plutôt enregistrer le code dans des fichiers.

On peut donc imaginer avoir un fichier header.php qui contient le code HTML de l'entête de la page, un fichier footer.php qui contient le code HTML du pied de page, et un fichier menu.php qui contient le code HTML du menu.
On peut alors inclure ces fichiers dans les pages du site en utilisant include ou require.

Voici un exemple simplifié d’utilisation de include : 

   
    <?php
        include 'header.php';
        include 'menu.php';
    ?>
    <h1>Page d'accueil</h1>
    <p>Bienvenue sur notre site !</p>
    <?php
        include 'footer.php';
    ?>
