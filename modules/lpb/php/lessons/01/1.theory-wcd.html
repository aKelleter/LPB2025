Dans cette première leçon, nous allons afficher à l'écran la chaîne de caractères : "Hello World !

Ou écrire du code PHP

De façon générale nous pouvons écrire du code PHP soit:

  • dans des fichiers dédiés, c’est-à-dire des fichiers qui ne vont contenir que du PHP et qui porteront l'extension '.php'.
  • nous intégrerons du code PHP au sein des fichiers HTML, mais qui porteront également l'extension '.php'.

La balise PHP

Le serveur mais plus particulièrement l'interprêteur PHP, pour être en mesure d’exécuter le code PHP, doit le reconnaitre. Pour lui indiquer qu'un bloc de code est écrit en PHP, nous allons entourer ce code avec une balise PHP qui a la forme suivante : 

    <?php ?>

Lorsqu’on intègre du PHP dans du code HTML, on peut intégrer du code PHP à n’importe quel endroit dans la structure HTML. On peut donc, placer un bloc de code PHP en dehors de l'élément html. Pour terminer, on va pouvoir déclarer plusieurs balises PHP à différents endroits.

La syntaxe de base

  1. Pour commencer on écrit la balise d'ouverture de PHP : <?php
  2. On écrit nos instructions, en l'occurence ici : echo 'Hello World !';
  3. Une instruction en PHP doit toujours se terminer par un point-virgule ';'. Si vous l’oubliez, votre script ne fonctionnera pas correctement.

Enregistrer et exécuter un fichier PHP

A partir du moment où l'on ouvre une balise PHP dans un fichier HTML, nous allons devoir enregistrer le fichier avec l’extension .php. C’est essentiel pour que le serveur comprenne que la page contient du code PHP qu’il devra exécuter.

Où enregistrer nos fichiers PHP sur notre ordinateur ?
Cela va dépendre de votre environnement de développement. Dans les faits vous aller devoir ou avez déjà installé un serveur web local (souvent Apache), un interprêteur PHP et un moteur de base de données (ou pas :)). Avec les environnements Xampp ou Mampp ce sera par défaut dans le répertoires htdocs, avec Xampp, c'est le répertoire www par défaut. C'est la configuration du serveur web qui va déterminer l'emplacement où nous allons enregistrer nos fichiers.

Bref, dans le cours j'utilise personnellement Xampp. Xampp est Cross Plateform (installable sur Windows, Linux, Macos). A l'inverse, Wamp est uniquement disponible sur Windows et Mamp sur Macos. Sur les machines des labo de l'Ifapme c'est Wamp qui est installé.

Pour accéder et exécuter vos fichiers PHP et autres. Vous utiliserez dans votre navigateur préféré (Firefox :)) une URL qui pourrait être structurée comme celle-ci : http://localhost/monsite/monfichier.php

NOTE : Lorsque vous essayez d'accéder à une ressource disponible sur un serveur web et plus particulièrement un répertoire sans indiquer de fichier (ex.: http://localhost/monprojet/), le comportement par défaut du serveur web va être de rechercher un fichier index.html ou index.php (ou .asp si IIS - serveur web de Microsoft). Si le fichier est trouvé il est retourné/interprêté vers le client (votre navigateur par ex) dans le cas contraire il adoptera un comportement en fonction de sa configuration.
