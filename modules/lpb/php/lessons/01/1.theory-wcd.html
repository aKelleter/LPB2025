Ou écrire du code PHP

De façon générale nous pouvons écrire du code PHP soit:

  • dans des fichiers dédiés, c’est-à-dire des fichiers qui ne vont contenir que du PHP et qui porteront l'extension '.php'.
  • nous intégrerons du code PHP au sein des fichiers HTML, mais qui porteront également l'extension '.php'.

La syntaxe de base

  1. Pour commencer on écrit la balise d'ouverture de PHP : <?php
  2. On écrit nos instructions, en l'occurence ici : echo 'Hello World !';
  3. Une instruction en PHP doit toujours se terminer par un point-virgule ';'. Si vous l’oubliez, votre script ne fonctionnera pas correctement.

La balise de fermeture de PHP est ?>.
Elle est utilisée pour indiquer la fin d'un bloc de code PHP.
Elle est facultative si le code PHP si le fichier ne contient que du code PHP.

Enregistrer et exécuter un fichier PHP

A partir du moment où l'on ouvre une balise PHP dans un fichier HTML, nous allons devoir enregistrer le fichier avec l’extension .php. C’est essentiel pour que le serveur comprenne que la page contient du code PHP qu’il devra exécuter.

Où enregistrer nos fichiers PHP sur notre ordinateur ?
Cela va dépendre de votre environnement de développement. Dans les faits vous aller devoir ou avez déjà installé un serveur web local (souvent Apache), un interprêteur PHP et un moteur de base de données (ou pas :)). Avec les environnements Xampp ou Mampp ce sera par défaut dans le répertoires htdocs, avec Xampp, c'est le répertoire www par défaut. C'est la configuration du serveur web qui va déterminer l'emplacement où nous allons enregistrer nos fichiers.

Vous trouverez ci-dessous quelques-uns des différents environnements de développement PHP. Des systèmes que l'on pourrait qualifier de clé sur porte.

  • Xampp. Xampp est Cross Plateform (installable sur Windows, Linux et Macos).
  • Wamp. Wamp est uniquement disponible sur Windows.
  • Mamp. Mamp est disponible à la fois sur Macos et sur Windows.

Personnellement, j'installe manuellement un environnement de développement PHP en configurant un serveur web local avec Apache, PHP et MySQL. Mais cette méthode demande un peu plus de connaissances techniques pour être mise en place correctement.

Pour accéder et exécuter vos fichiers PHP et autres ressources, vous accéderez par le biais de votre navigateur à une URL qui pourrait être structurée telle que celle-ci :
http://localhost/monsite/monfichier.php

NOTE : Lorsque vous essayez d'accéder à une ressource disponible sur un serveur web et plus particulièrement un répertoire sans indiquer de fichier (ex.: http://localhost/monprojet/), le comportement par défaut du serveur web va être de rechercher un fichier nommé index.html ou index.php (ou .asp si IIS - serveur web de Microsoft). Si le fichier est trouvé il est d'abord interprêté et le résultat est retourné vers le client (votre navigateur par ex) dans le cas contraire il adoptera un comportement en fonction de sa configuration.
