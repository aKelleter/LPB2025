I. La portée des variables en PHP

Nous allons aborder dans cette leçon un concept important en programmation, la portée des variables. La portée d'une variable, c'est l'endroit où celle-ci est accessible. En PHP, il existe trois types de portée de variables : les variables locales, les variables globales et les variables statiques.

En PHP, nous pouvons déclarer des variables n’importe où dans notre script : au début du script, à l’intérieur de boucles, au sein de nos fonctions, etc. Je n'ai pas dit que c'était une bonne idée, mais c'est possible.

La "règle" de base est que l’endroit où l'on déclare une variable va déterminer l’endroit où la variable va être accessible. L'accessibilité d'une variable est appelée la portée de la variable. Toute variable définie en dehors d’une fonction a une portée globale. Par définition, une variable qui a une portée globale est accessible dans tout le script sauf dans les espaces locaux d’un script.

Les variables déclarées à l'intérieur d'une fonction ont une portée locale. Cela signifie que ces variables ne sont accessibles que dans la fonction où elles ont été déclarées. Une variable locale est détruite dès que la fonction dans laquelle elle a été déclarée se termine.

Explications de la portée des variables en PHP avec l'Exo1 du code source

Au début du script, nous déclarons une variable $var en dehors de toute fonction. Notre variable possède donc une portée globale. Dans notre première fonction fnport1(), on tente d’afficher le contenu de notre variable $var déclarée globalement. Cela ne va pas fonctionner, en effet, une variable globale n’est par défaut pas accessible dans un espace local.

Dans notre seconde fonction fnport2(), nous définissons une variable $var. C'est donc une variable locale à la fonction. Ici on déclare et initialise $var pour ensuite l'afficher avec un "echo". Il est important de comprendre que les deux variables $var, c'est-à-dire , la variable globale déclarée et intialisée en début de script et la variable $var locale de la fonction fnport2() sont deux variables différentes (elles possèdent des adresses mémoires différentes). C'est ce que l'on constate lorsque l’on affiche ensuite le contenu de notre variable $var globale (=100) qui n’a pas été influencée par son homologue local (=5).

Notre troisième fonction fnport3() définit quand à elle une variable $compteur = 0, qui est ensuite incrémentée et retournée. Si on appelle plusieurs fois fnport3(), on se rend compte que le résultat est toujours 1. Cela s’explique par le fait que la variable est détruite à la fin de l’exécution de chaque fonction et est donc réinitialisée sur sa valeur $compteur = 0 à chaque fois qu’on appelle la fonction.

Finalement, notre quatrième fonction fnport4() définit une variable $nb. Lorsqu’on essaie d’afficher le contenu de $nb depuis l’espace global, aucune valeur n’est retournée puisqu’une variable définie localement n’est accessible que dans l’espace dans laquelle elle a été définie par défaut.

II. Accéder à une variable de portée globale depuis un espace local

Ce n'est pas conseillé, mail il arrive parfois que l'on souhaite accéder à des variables possédant une portée globale (c’est-à-dire définies en dehors d’une fonction) dans le code à l’intérieur d’une fonction.
Pour cela, on va pouvoir utiliser le mot clef global avant la déclaration des variables qu’on souhaite utiliser dans notre fonction. Cela va nous permettre d’indiquer que les variables déclarées dans la fonction sont en fait des références aux variables globales.
Attention : si on modifie la valeur de ces variables dans notre fonction, la valeur des variables globales sera également modifiée.

Si vous avez besoin d'accéder à une variable définie localement depuis l’espace global, vous devrez utiliser le mot clef return pour retourner la valeur de la variable locale. Une variable locale n’aura toujours qu’une portée locale, sa portée ne pourra pas être étendue dans l’espace global.

IV. Le mot clef static

Comme nous l'avons vu précédemment, une variable définie localement va être supprimée ou détruite dès la fin de l’exécution de la fonction dans laquelle elle a été définie.
En PHP, il est possible de déclarer des variables statiques. Une variable statique est une variable qui conserve sa valeur entre les appels successifs d’une fonction.
Pour déclarer une variable statique, on utilise le mot clef static avant la déclaration de la variable.

Note : lorsque nous initialisons une variable en utilisant static, la variable ne sera initialisée que lors du premier appel de la fonction (dans le cas contraire, le mot clef static n’aurait pas grand intérêt).
