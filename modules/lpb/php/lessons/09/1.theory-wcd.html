Dans cette leçon, nous allons étudier et comprendre l’intérêt des structures de contrôle conditionnelles en PHP. Une structure de contrôle conditionnelle est un ensemble d’instructions qui permet de contrôler l’exécution d'une partie du code.

Les structures de contrôle conditionnelles (ou plus simplement conditions) vont nous permettre d’exécuter différents blocs de code selon qu’une condition spécifique soit vérifiée ou pas. On va pouvoir utiliser les conditions pour, par exemple, afficher certains éléments (ou parties) sur notre site selon que l’utilisateur soit identifié ou un simple visiteur.

Nous allons très souvent utiliser les conditions avec des variables. En fonction de la valeur stockée dans une variable, nous allons vouloir exécuter un bloc de code plutôt qu’un autre. Les conditions être obligatoires pour rentre un site dynamique en fonction du contexte.

En PHP, nous allons pouvoir utiliser les conditions suivantes :

  • La condition if (si) ;
  • La condition if… else (si… sinon) ;
  • La condition if… elseif… else (si… sinon si… sinon).
Etudions chacune de ces conditions :) Petit rappel, pour pouvoir travailler avec des conditions nous allons très souvent utiliser les opérateurs de compararaisons. Veuillez relire la leçon sur les opérateurs et plus particulièrement la section sur les opérateurs de comparaisons.

Rappel

Lorsqu’on utilise un opérateur de comparaison, le PHP va comparer la valeur à gauche de l’opérateur à celle à droite (les opérandes). On dit également qu’il évalue la comparaison.

Si la comparaison est vérifiée ou validée, alors le PHP renvoie la valeur booléenne true. Si le test de comparaison échoue, alors PHP renvoie la valeur booléenne false. La compréhension de cette notion est très importante, car nos conditions vont nous permettre d'exécuter ou non le bloc de code qui les suivent.

I. L'instruction If

L'instruction if est une des plus importantes instructions de tous les langages, PHP inclus. Elle permet l'exécution conditionnelle d'une partie de code. Les fonctionnalités de l'instruction if sont les mêmes en PHP qu'en C : 

        if (expression)
            commandes

La condition if est également la plus simple, puisqu’elle va nous permettre d’exécuter un bloc de code si et seulement si le résultat d’un test vaut true
Comme nous l'avons vu dans la leçon consacrée aux expressions, l'expression est convertie en sa valeur booléenne. Si l'expression vaut true, PHP exécutera l'instruction et si elle vaut false, l'instruction sera ignorée.

Souvent, vous voulez que plusieurs instructions soient exécutées après un branchement conditionnel. Bien évidemment, il n'est pas obligatoire de répéter l'instruction conditionnelle if autant de fois que vous avez d'instructions à exécuter. À la place, vous pouvez rassembler toutes les instructions dans un bloc {...}.

Pour terminer, vous pouvez imbriquer indéfiniment des instructions if dans d'autres instructions if, ce qui permet une grande flexibilité dans l'exécution d'une partie de code suivant un grand nombre de conditions.

II. L'instruction If else

Souvent, vous voulez exécuter une instruction si une condition est remplie et une autre instruction si cette condition n'est pas remplie. C'est à cela que sert else. else fonctionne après un if et exécute les instructions correspondantes au cas où l'expression du if est évaluée à false.

Traduite en français l'instruction if else signifie : Si la condition est vraie, alors exécute le bloc de code A, sinon exécute le bloc de code B.

IMPORTANT : Attention, l'instruction else doit être aussi comprise comme : "dans tous les autres cas". Cela signifie que si la condition du if est évaluée à false, alors le bloc de code du else sera exécuté.

III. L'instruction If elseif else

La condition if elseif else (« si sinon si sinon ») est une structure conditionnelle encore plus complète que la condition if else. De fait, elle nous permet de générer autant de cas que ce que l’on souhaite. En effet, nous allons pouvoir écrire autant de elseif que l’on veut dans notre condition if elseif else, chacun avec un test différent.

Comme on peut le constater, les elseif occupent un rôle similaire au if de départ dans le sens où chacun d’entre eux va posséder son propre test, qui bien évidemment est obligatoire.

Notez que dans le cas où plusieurs elseif possèdent un test qui va être évalué à true, seul le code du premier elseif rencontré sera exécuté.
En effet, dès qu’un test va être validé, le PHP va ignorer les tests suivants.
Notez également qu’on devra toujours obligatoirement terminer notre condition if…elseif…else avec un else qui servira à gérer toutes les issues (ou les cas) non pris en charge par le if ou par les elseif.

IV. Des conditions avec plusieurs comparaisons

Nous avons vu dans une leçon précédente les opérateur logiques AND, &&, OR, ||, XOR. Ces derniers vont nous permettre de créer des conditions avec plusieurs comparaisons. On va ainsi pouvoir choisir d’exécuter un code uniquement si toutes les comparaisons valent true, ou si au moins l’une d’entre elles vaut true, ou encore si seulement l’une d’entre elles vaut true.

Voir l'exemple : Ex 6 , dans le code source.

Rappel

Avec l'opérateur && / AND. Chacune des deux comparaisons va devoir être évaluée à true pour que le code de la condition soit exécuté.
Avec l’opérateur || / OR. Dans ce cas, il suffit qu’une seule comparaison soit évaluée à true pour que le code de la condition soit exécuté. Mais toutes les évalulations peuvent être évaluée à true
Avec l’opérateur XOR. Dans ce cas, ne doit y avoir qu’une seule comparaison évaluée à true pour que le code de la condition soit exécuté (OU exclusif).

On peut utiliser plusieurs opérateurs logiques dans une même condition pour ajouter des règles et des comparaisons à celle-ci. Il faudra ici faire attention à la priorité des différents opérateurs pour obtenir le résultat voulu ou utiliser des parenthèses pour forcer la priorité de certains opérateurs par rapport à d’autres. De toute façon, il est toujours préférable de mettre des parenthèses pour plus de clarté. (ex : AND possède une priorité plus importante que XOR) 

    if (condition1 && condition2 || condition3) {
        // code à exécuter
    }

    if (condition1 && (condition2 || condition3)) {
        // code à exécuter
    }

V. Inverser la valeur logique d’une variable, d’une condition ou d’un test

Il est possible d’inverser la valeur logique d’une variable, d’une condition ou d’un test en utilisant l’opérateur ! (négation). Cet opérateur va permettre de passer d’une valeur true à false et inversement.

Voir l'exemple : Ex 7 , dans le code source.

Rappel

Lors d’une conversion en booléen, seules les valeurs suivantes sont évaluées à false :

  • Le booléen false lui-même ;
  • Le chiffre 0 ;
  • La chaine de caractères vide et la chaine de caractères « 0 » (zéro) ;
  • Le type NULL ;
  • Un tableau vide.
Toute autre valeur est évaluée à true.

Un exemple concret:

    $var1 = 4; 
    $var2 = -12;
    
    if(!$var1 >= 0 AND !$var2 >= 0){
        echo 'Ce texte est toujours affiché !
';
    }

Explication rapide :
$var1 = 4, pour appliquer ! le PHP transtype en booléen, 4 == true, l'inverse de true devient false, false redevient (transtypage) 0, 0 >= 0 est vrai.
$var1 = -12, pour appliquer ! le PHP transtype en booléen, -12 == true, l'inverse de true devient false, false redevient 0 (transtypage), 0 >= 0 est vrai.
Evaluation de (true AND true) = true, le code est exécuté.
