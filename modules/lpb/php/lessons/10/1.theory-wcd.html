I. L'opérateur ternaire : ?:

Pour présenter l'opérateur ternaire nous allons commencer par un exemple simple de condition : 

    if ($a) {
        $b = $a;
    } else {
        $b = 1;
    }

Ce qui signifie : si la variable $a est vraie, alors la variable $b prendra la valeur de $a, sinon elle prendra la valeur 1.

Et maintenant si je vous disais que le code ci-dessous est équivalent à celui du dessus : 

    $b = $a ? $a : 1;
    $b = ($a) ? $a : 1; // Pour plus de clarté

L'opérateur ternaire permet de réduire notre condition en une seule ligne. Les structures ternaires vont se présenter sous la forme suivante :
test ? code à exécuter si true : code à exécuter si false
$var = [IF] ? [THEN] : [ELSE];

Encore plus court ? Oui, c'est possible !

Ce raccourci permet d’omettre la partie centrale de l’opérateur ternaire. Il est toutefois à manier avec précaution car il utilise quelques subtilités du langage PHP.
$var = [IF] ?: [ELSE];
En utilisant cette écriture, c’est la partie située avant l’opérateur qui sera renvoyée si le test est évalué à true et la partie après l’opérateur sinon.

Ici, il faut bien comprendre que PHP ne va pas renvoyer le contenu de $var mais le résultat de la comparaison $var >= 0, c’est-à-dire true, convertie sous forme d’entier (c’est-à-dire 1).

II. L'opérateur de fusion null ou de coalescence null : ??

L’opérateur fusion null ?? ressemble dans sa syntaxe et dans son fonctionnement à l’opérateur ternaire.
Cet opérateur utilise la syntaxe suivante : test ?? code à renvoyer si le résultat du test est NULL.

Si l’expression à gauche de l’opérateur vaut NULL, alors l’expression à droite de l’opérateur sera renvoyée. Dans tous les autres cas, c’est l’expression à gauche de l’opérateur qui sera renvoyée. L’opérateur fusion null est un opérateur de comparaison qui permet de simplifier l’écriture de conditions. Il est apparu avec PHP 7.0.0. Il est très utile pour simplifier le code et éviter les erreurs de type Undefined variable.

IMPORTANT, cet opérateur n'émet pas de notice ou avertissement si la partie gauche n'existe pas, exactement comme isset(). Ceci est particulièrement utile pour les clés des tableaux.
L'opérateur de fusion null à une précédence faible. Ceci signifie que pour le mélanger avec d'autres opérateurs, des parenthèses seront requises (tel que la concaténation de chaînes ou les opérateurs arithmétiques).

On va généralement utiliser cet opérateur pour tester si une variable contient quelque chose ou pas et, dans le cas où la variable est vide, lui faire stocker une valeur.

Dans le second exemple, on teste nos variables $x, $y et $z en utilisant l’opérateur fusion null.
Voir le second exemple de l'Ex4 du code source. 

        $x = 5; 
        $y = NULL;
        $z;
        
        $resultx = $x ?? 'NULL';
        $resulty = $y ?? 'NULL';
        $resultz = $z ?? 'NULL';
Si la variable est vide ou contient NULL, la valeur NULL sera renvoyée et stockée dans $resultx, $resulty ou $resultz. Dans le cas contraire, c’est le contenu de la variable qui sera stocké.
