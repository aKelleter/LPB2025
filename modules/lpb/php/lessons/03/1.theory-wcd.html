Comment et pourquoi utiliser echo ou print dans votre code

En PHP, il y a deux façons d'obtenir une sortie (la sortie standard par défaut c'est l'écran) : echo et print.

Ces deux commandes servent à envoyer du texte, des nombres ou du code HTML vers la sortie (souvent le navigateur web).

La commande echo

echo est une instruction (et non une fonction). Elle permet d’afficher une ou plusieurs valeurs.

La commande echo est utilisée pour afficher une ou plusieurs chaînes de caractères. Elle peut prendre plusieurs arguments séparés par des virgules.

Différences entre echo et print

Différence echo print
Type Instruction Expression
Retourne une valeur ? ❌ Non ✅ Oui (1)
Peut afficher plusieurs valeurs ? ✅ Oui (séparées par des virgules) ❌ Non
Vitesse Légèrement plus rapide Légèrement plus lente
Usage courant Très fréquent Moins fréquent

Bonnes pratiques

  • Utilisez echo pour la plupart des affichages.
  • Réservez print aux cas où vous avez besoin de retourner une valeur (rare).

Pour éviter les erreurs :

  • Entourez les chaînes de guillemets "...".
  • Fermez correctement les balises PHP <?php ... ?>.

Avec des parenthèses ?

Vous pouvez mettre des parenthèses après echo ou print, mais ce n’est pas obligatoire.
Avec echo, les parenthèses sont limitées à un seul argument.
Avec print, c’est toujours autorisé, car print agit comme une fonction.
Evitez d'utiliser des parenthèses, elles n'apportent pas de valeur ajoutée.

La version courte d'echo

echo dispose aussi d'une syntaxe courte, où vous pouvez faire suivre immédiatement la balise PHP ouvrante d'un signe égal (=).
Cette syntaxe est souvent utilisée sur les sites ou applications qui mélange le code HTML et le code PHP.

Le Heredoc

Le Heredoc permet d’écrire de longs textes multi-lignes sans avoir à gérer les guillemets ni les concaténations.
C’est très pratique pour :

  • du HTML généré par PHP
  • générer des requêtes SQL complexes
  • générer des blocs de texte formatés (ex. : modèles d’e-mail, JSON, XML,...)
  • des chaînes longues,
  • etc.

  • La ligne de fermeture doit être suivie d’un point-virgule ;

Bonus le Nowdoc

Le Nowdoc est similaire au Heredoc, mais il traite le contenu comme une chaîne littérale.
Cela signifie que les variables et les séquences d’échappement ne sont pas interprétées.
