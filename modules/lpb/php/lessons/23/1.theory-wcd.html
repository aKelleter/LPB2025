I. Présentation

Les Superglobales sont les variables internes du langage PHP, elles sont toujours disponibles, quel que soit le contexte. Ces variables vont être accessibles n’importe où dans le script et quel que soit le contexte, qu’il soit local ou global.

Il existe 9 superglobales. Ces variables sont toutes des tableaux. La plupart des scripts PHP vont utiliser les variables superglobales car ces dernières vont s’avérer très souvent très utiles . Il est donc indispensable de bien les connaitre et de savoir les manipuler.

Les variables superglobales PHP sont les suivantes :

  • $GLOBALS
  • $_SERVER
  • $_REQUEST
  • $_GET
  • $_POST
  • $_FILES
  • $_ENV
  • $_COOKIE
  • $_SESSION

On écrira toujours les superglobales en majuscules. C'est une convention qui permet de les différencier des variables dites « classiques » que nous créons nous-mêmes. Par ailleurs, vous pouvez remarquer que toutes les superglobales à l’exception de $GLOBALS commencent par un underscore _.

II. La superglobale $GLOBALS

La superglobale $GLOBALS est un tableau associatif qui contient toutes les variables globales du script PHP. La variable superglobale $GLOBALS est un peu différente des autres superglobales puisque c’est la première superglobale qui a été créée en PHP. Cette superglobale va nous permettre d’accéder à des variables définies dans l’espace global depuis n’importe où dans le script et notamment depuis un espace local (dans une fonction).

III. La superglobale $_SERVER

La superglobale $_SERVER est un tableau associatif qui contient des informations sur le serveur et l’environnement d’exécution du script PHP. Cette superglobale va nous permettre de récupérer des informations sur le serveur, sur le client, sur le script en cours d’exécution, sur les fichiers, etc. Cette superglobale est très souvent utilisée pour récupérer des informations sur le serveur ou sur le client.

VIII. La variable superglobale $_COOKIE

La superglobale $_COOKIE est un tableau associatif qui contient les cookies envoyés par le client au serveur. Cette superglobale va nous permettre de récupérer les cookies envoyés par le client au serveur. Les cookies sont des fichiers textes qui sont stockés sur le poste client et qui sont envoyés au serveur à chaque requête HTTP. Cette superglobale est très souvent utilisée pour récupérer les cookies envoyés par le client au serveur.

La superglobale $_SESSION est un tableau associatif qui contient les variables de session du client. Cette superglobale va nous permettre de stocker des variables de session du client. Les variables de session sont des variables qui sont stockées sur le serveur et qui sont accessibles par le client. Cette superglobale est très souvent utilisée pour stocker des variables de session du client.
