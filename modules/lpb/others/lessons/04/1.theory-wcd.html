1. Préambule

Comprendre comment PHP peut choisir quel bloc HTML afficher en fonction d’une variable, grâce à une structure conditionnelle if / else.

2. Principe général

PHP est un langage côté serveur :

  • Il décide avant l’affichage ce qui doit apparaître dans la page HTML.
  • Donc, si une condition est vraie, un bloc HTML sera envoyé au navigateur.
  • Sinon, un autre bloc HTML sera envoyé.

Exemple commenté : afficher deux formulaires différents

3. Comment expliquer cela simplement ?

1. PHP décide AVANT d’envoyer la page au navigateur

Le navigateur ne voit qu’un seul des deux formulaires.
Il ne voit pas le if, else ou endif.

C’est PHP qui choisit quel formulaire envoyer.

2. Si la variable $estConnecte est vraie (true)

PHP envoie le formulaire de déconnexion.

3. Sinon, si la variable $estConnecte est fausse (false)

PHP envoie le formulaire de connexion.

5. Conclusion

  • On peut mélanger PHP et HTML.
  • Les conditions permettent d’afficher différents contenus.
  • PHP choisit quoi envoyer, le navigateur ne voit que le résultat final.
  • C'est très utilisé : formulaires, menus, connexion, jeux, etc.

6. Utilise-t-on encore ce type d'écriture dans le code professionnel en PHP ?

1. Oui, c’est encore utilisé — surtout dans les petites applications et les cours

  • des projets PHP simples
  • des scripts avec un seul fichier
  • du code pédagogique
  • des petites pages PHP sans framework
  • des sites sans moteur de template

est parfaitement normal et professionnel.

2. Mais dans les “vrais” gros projets professionnels, on utilise autre chose

Quand le projet devient plus grand, on sépare plus strictement :

  • la logique (PHP)
  • la présentation (HTML)

On utilise alors un moteur de templates :

  • Twig (Symfony, Drupal…)
  • Blade (Laravel)
  • Plates, Mustache, etc.

3. Pourquoi on utilise ces templates dans les grands projets ?

  1. Séparer proprement la logique PHP et l’affichage HTML
    2. → meilleur code, plus lisible, plus facile à maintenir.
  2. Les fichiers HTML sont propres
    3. → pas de <?php ?> partout.
  3. On protège des erreurs
    4. → le moteur empêche d’exécuter du PHP dans la vue (sécurité).
  4. Plus facile pour travailler en équipe (développeurs + designers)
