1. Introduction

Au début du module LPB nous avons installé sur nos machines un serveur web (Apache), un serveur de base de données (MariaDB) et un serveur PHP en installant le logiciel XAMPP. Cet outil a également installé un outil de gestion de base de données de type "web based" nommé phpMyAdmin. Nous ajouterons un autre outil de gestion de base de données de type "desktop application" nommé DBeaver ou Mysql Workbench pour celles et ceux qui le préfèrent. Mais ce cours ne traitera que de DBeaver.

Un petit mot sur DBeaver. C'est un outil de gestion de base de données universel et gratuit qui prend en charge la plupart des bases de données relationnelles telles que MySQL, MariaDB, PostgreSQL, SQLite, etc. Il est basé sur Eclipse et est disponible sous licence publique générale GNU.

Installation de DBeaver

Pour installer DBeaver, il suffit de se rendre sur le site officiel de l'application https://dbeaver.io/download/ et de télécharger la version correspondant à votre système d'exploitation. Pour les utilisateurs de Windows, il suffit de télécharger le fichier .exe et de l'exécuter. Pour les utilisateurs MacOS, il suffit de télécharger le fichier .dmg et de l'installer. Pour les utilisateurs de Linux, il suffit de télécharger le fichier .deb ou .rpm

Maintenant que nous avons installé tous les outils graphiques nécessaires pour gérer nos bases de données, nous allons voir comment créer une base de données et une table en utilisant le terminal de votre système d'exploitation. :D
Ensuite, nous verrons comment faire la même chose en utilisant phpMyAdmin et ainsi qu'un logiciel graphique tel que DBeaver. Vous aurez ainsi une idée de la manière dont les bases de données sont gérées à travers différents outils.

2. Vérification de l'installation et du bon fonctionnement du système SGBD installé sur votre ordinateur

Pour se faire lancer le programme XAMPP :

  • Sur Windows, cliquez sur le raccourci sur le bureau ou dans le menu démarrer
  • Sur MacOS, ouvrez le launchpad, rechercher manager-osx et lancer l'application ou encore à l'aide du terminal, tapez sudo /Applications/XAMPP/xamppfiles/xampp start ou pour terminer lancer l'application XAMPP dans le dossier Applications
  • Sur Linux, ouvrez le terminal et tapez sudo /opt/lampp/lampp start

Une fois le programme lancé, vous devriez voir une interface ressemblant à celle ci-dessus. Cliquez sur le bouton Start pour démarrer les serveurs Apache et MySQL. Si tout se passe bien, vous devriez voir les deux serveurs démarrés comme sur l'image ci-dessous.

Si besoin est vous pouvez modifier les ports sur lesquels les serveurs écouteront les requêtes. Pour ce faire, cliquez sur le bouton Config et sélectionnez Port. Par défaut, Apache écoute sur le port 80 et MySQL sur le port 3306. Vous pouvez les modifier si vous avez des conflits avec d'autres applications.

Si les témoins à côté des serveurs sont de couleurs vertes, c'est que ces derniers ont démarré sans encombre. Dans le cas contraire, il faut chercher la raison pour laquelle les serveurs n'ont pas démarré correctement.

Si tout est en ordre, nous pouvons passer à la suite. :D
