1. Introduction

L'algèbre relationnelle est un langage formel utilisé pour manipuler et interroger les bases de données relationnelles. Elle fournit un ensemble d'opérations permettant de définir des requêtes pour extraire ou transformer des données stockées sous forme de relations (tables).

2. Concepts Fondamentaux

Relation et Schéma Relationnel

  • Relation : Une table contenant des tuples (lignes) et des attributs (colonnes).
  • Schéma relationnel : La structure d'une relation, définie par un ensemble d'attributs et leur domaine
  • Instance d'une relation : L'ensemble des tuples présents dans la table à un instant donné.

Les Opérations de l'Algèbre Relationnelle

Les opérations de l'algèbre relationnelle se divisent en deux catégories : les opérations de base et les opérations dérivées.

Les Opérations de Base
  1. Sélection (σ - sigma) : Filtre les lignes d'une relation selon une condition.
    • Notation : σ condition (R)
    • Exemple : σ age>30 (Employés) retourne les employés ayant plus de 30 ans.

  2. Projection (π - pi) : Extrait des colonnes spécifiques d'une relation.
    • Notation : π attributs (R)
    • Exemple : π nom, salaire (Employés) retourne uniquement les noms et salaires des employés

  3. Union (∪) : Combine les tuples de deux relations de même schéma.
    • Notation : R ∪ S
    • Exemple : EmployésFrance ∪ EmployésAllemagne retourne la liste des employés en France et en Allemagne.

  4. Différence (−) : Retourne les tuples de la première relation qui ne sont pas dans la seconde.
    • Notation : R - S
    • Exemple : EmployésFrance - EmployésAllemagne retourne les employés en France qui ne sont pas en Allemagne.

  5. Produit Cartésien (×) : Combine tous les tuples de deux relations.
    • Notation : R × S
    • Exemple : Employés × Départements génère toutes les combinaisons possibles d'employés et de départements.

  6. Jointure (⋈) : Combine des relations sur la base d'une condition commune.
    • Notation : R ⋈condition S
    • Exemple : Employés ⋈Employés.id = Départements.chef_id Départements
Les Opérations Dérivées
  1. Intersection (∩) : Retourne les tuples présents dans les deux relations.
    • Notation : R ∩ S
    • Exemple : EmployésFrance ∩ EmployésAllemagne donne les employés travaillant en France et en Allemagne.

  2. Division (/) : Identifie les éléments d'une relation qui sont associés à tous les éléments d'une autre relation.
    • Notation : R / S
    • Exemple : Trouver les étudiants ayant suivi tous les cours d'un programme.

3. Conclusion

L'algèbre relationnelle est essentielle pour comprendre le fonctionnement des SGBDR et optimiser les requêtes SQL. Elle sert de base théorique à SQL, qui implémente ces opérations sous forme de requêtes exploitables dans les bases de données relationnelles modernes.
